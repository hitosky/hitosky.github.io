ES6技巧

1.强制参数传入默认值

ES6规定只有当参数在被使用的时候，才会被“计算”值。利用这点可以检查用户是否传入该参数。

  function mandatory() {
      throw new Error('Missing parameter');
    }
  //当用户未提供参数时会调用mandatory()
  function foo(mustBeProvided = mandatory()) {
      return mustBeProvided;
    }

上述foo只有在mustBeProvided未提供时才会调用mandatory()。 >foo() Error:Missing parameter >foo(1) 1

2.通过for-of遍历数组

ES5中遍历数组常采用forEach的方法，但是forEach方法不能break，也无法return； 采用for-in的方法，如for(var x in arr),x是对应的下标，并不是数组元素。 ES6迎来了for-of方法，允许直接访问数组元素：

    var arr = [1,2,3];
    for(var x of arr)
    	console.log(x);
    /*1
      2
      3*/

3.Rest参数

rest参数根据字面可以理解为“剩余参数”,开头为省略号，类似于...restParas，rest参数的意义在于，将不确定数量的参数放入到一个数组中，以此来区分确定传入的参数。 现在有一函数如下：目的是为了实现参数x与其他参数的累加和

function add(x){
    for(var i=1;i<arguments.length;i++)
        x+=arguments[i];
    return x;
}

上面是传统的做法，这种做法有一个坏处就是确定要传入的参数x与未知数量的其他参数混合在同一个数组中，其他参数下标从1开始，如果想在x之前或之后添加几个确定传入的参数就要修改内部循环。 再看使用rest参数的做法：

function add(x,...rest){
    for(var i=0;i<rest.length;i++)
        x+=rest[i];
    return x;
}

这样无论x前后参数序列怎么变，都不会影响rest数组与内部循环

4.Destructuring解构赋值

ES6这个新特性很像python中的赋值，就是简单直接地将某个集合中元素值赋给另一个集合中相同位置或下标的元素。 过去交换两个元素，需要

var a=1;
var b=2;
a+=b;
b=a-b;
a=a-b;

现在只要简单的

var a=1;
var b=2;
[a,b]=[b,a];
