BFC块级上下文

之前一直不清楚BFC是什么鬼，今天特意看了下大牛们的博文，这里总结下。

网页在浏览器器中通过CSS布局的基本元素是盒模型Box，浏览器在渲染Box的时候，是根据其格式化上下文FC来进行渲染的，换句话说，就是Box内部元素位置如何排列以及各元素之间的关系与作用均由这个Box的FC所决定。这种做法有点类似js中的执行上下文EC来决定变量和对象的取值。

FC的分类

在CSS2.1中格式化上下文只有BFC和IFC（内联格式化上下文）, CSS3中还增加了GFC（网格布局格式化上下文）和FFC（自适应格式化上下文）。

FC的决定因素

FC类别主要由Box的display所决定，其他属性如float、position、overflow也能影响FC。

哪些Box能形成BFC

以下条件至少满足其一就能形成BFC

  1. float 的值不为 none
  2. position 的值为 fixed 或 absolute
  3. display 的值为 table-cell、table-caption、inline-block、flex 或 inline-flex
  4. overflow 的值不为 visiable
  5. fieldset元素

BFC内部元素特性

  • 内部Box会在垂直方向，从顶部开始一个接一个地放置
  • Box垂直方向的距离由margin属性决定，属于同一个BFC的两个相邻Box的垂直margin会发生叠加
  • 对于从左往右顺序排列，每个元素的margin box的左边与包含块border box的左边重合；对于从右往左顺序则box右边重合。对于line box与浮动元素之间可能会产生文字环绕现象
  • BFC的区域不会与BFC外部的float box叠加
  • BFC内部子元素与BFC外部元素互相独立，互不影响
  • 计算BFC高度时，浮动元素也参与计算

BFC带来的问题

  • 外边距折叠问题
  • 文字环绕

BFC特性能为我们做什么

  • 闭合浮动
  • 多列页面布局

如何解决BFC特性对内部元素带来的‘负面’影响

基本思路就是将一个BFC中的内容，放到不同的BFC中
