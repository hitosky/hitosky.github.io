JS this指向

this常见的5种调用：

  1. 全局范围内 this=global
  2. 函数调用 this=global
  3. 对象方法的调用 this=object of calling method
  4. 构造函数的调用 this=object of constructor
  5. 显式地设置this（如apply调用） this=caller of apply

this值是在进入上下文的时候决定的，代码运行时其值不允许更改。

activeExecutionContext = {
  VO: {...},
  this: thisValue
};

普通的函数调用中，this的值由调用该函数的父作用域（调用表达式）决定。

When a function of an object was called, the object will be passed into the execution context as this value

会引起this改变的调用表达式也称引用类型，其形式类似：

var valueOfReferenceType={
  base:<base object>,
  propertyName:<property name>
};

this=valueOfReferenceType['base'];

引用类型的值只在两种情况下存在：

  1. 处理标识符：变量名，函数名，函数参数名，全局对象属性名
  2. 属性访问器：.运算符与[]运算符

如果(…)左边是一个引用类型，那么this=valueOfReferenceType['base']。其他情况下this的值一直是null,由于this=null无意义，所以this会被隐式转换为global。ES5严格模式下，this=undefined

确定this的值关键是得确定当前上下文的调用者

如果想快速判断this的指向，可以参考知乎某位大虾提出的在ES5下面的this判断流程：

  1. 函数是否是new调用？若是，则this指向新对象
  2. 函数是否是bind方法返回的？若是，则this指向指定对象。
  3. 函数是否通过apply/call调用？若是，则this指向指定对象。
  4. 是否作为对象的方法调用？若是，则this指向该对象
  5. this指向全局
